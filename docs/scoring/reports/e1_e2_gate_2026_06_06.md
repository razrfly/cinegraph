## Pre-registration (committed before the gate)

### E1 — `ixn_rt_<genre>` (8 codes)

- **Hypothesis:** the model's dominant tomatometer weight conflates genres — RT 92 on horror
  (Final Destination Bloodlines) and RT 93 on prestige drama (The Brutalist) read identically.
  Genre×RT interactions let the fit re-weight RT per genre.
- **Mechanism:** `genre_indicator × normalized_tomatometer` ∈ [0,1]; 0.0 = no genre / no RT
  (no signal, per the DerivedFeatures emit-0.0 convention).
- **Genre pick (pre-registered):** top-5 member-frequent narrative genres across the six served
  lists (drama 1146 · comedy 389 · romance 364 · crime 235 · thriller 169 — documentary excluded:
  rarely competes in prediction pools) + the 3 RT-inflation genres from the founding complaint
  (horror, action, science_fiction). Frequency table from the recorded query of 2026-06-06.
- **Falsifier:** Δ PR-AUC < 0.005 on > half the lists at `--sample 25000 --seed 1337`.
- **Known risk:** partial redundancy with the existing `genre_*` mains + the RT main — that
  redundancy is exactly what the gate measures.

### E2 — `rt_meta_gap` (1 code)

- **Hypothesis:** genre inflation shows up as RT ≫ Metacritic (Smile 2: 86 vs 67 → 0.19;
  The Brutalist: 93 vs 90 → 0.03). One cheap prestige-vs-crowd-pleaser axis.
- **Mechanism:** `max(rt − meta, 0)` over catalog-normalized values. Encoding CORRECTED from the
  Implementation Notes' 0.5-centered form, which would have made a missing source read as extreme
  meta-skew under the module's emit-0.0 convention (see #1081 comment, 2026-06-06).
- **Falsifier:** same as E1.

## Lessons

1. **E2's null has a structural explanation we should have seen coming:** a linear model with
   `rotten_tomatoes_tomatometer` and `metacritic_metascore` as separate features already spans
   every linear combination of them — including `rt − meta`. The only new content in
   `max(rt − meta, 0)` was the clamp, and the clamp adds nothing. **Rule for future specs: a
   derived feature must be NON-representable as a linear combination of existing features, or
   it's dead on arrival.** (E1's interaction passes that rule and still died — see 2.)
2. **E1 hurts the strong lists** (afi −0.124, tspdt −0.054, NFR −0.025): 8 extra RT-correlated
   features dilute small undersampled fits — the same dilution mechanism as the #1051 ablation
   finding. The only lists it helped (cult, both Sight & Sound) are the *unserved* taste lists,
   where there is no model to improve.
3. **Both features stay built + catalogued + `is_available: false`** (the indicator-family
   precedent: built, vetted, inert — re-testable if the model class ever changes, e.g. a tree
   model where interactions matter differently).
